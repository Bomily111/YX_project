# 隧道施工数字孪生平台 — 架构分析

## 技术栈概览

| 层级 | 技术选型 |
|------|---------|
| **前端框架** | Vue 3 (Composition API, `<script setup>`) + TypeScript |
| **构建工具** | Vite 7 + vue-tsc |
| **3D 地球引擎** | Cesium 1.140 (通过 vite-plugin-cesium 加载) |
| **UI 组件库** | Element Plus + 自定义科技风 UI |
| **状态管理** | Pinia |
| **路由** | Vue Router 4 |
| **图表** | ECharts 6 |
| **网络请求** | Axios |
| **数学/几何** | gl-matrix, @turf/turf |
| **事件总线** | mitt |
| **后端** | Express 5 + Multer (文件上传) |
| **后端脚本** | Python (TEM 数据处理) + C++ EXE/DLL (GPR 等) |

## 核心流程

### 启动流程

```
main.ts
  → AppConfig.loadConfig('/config/Application.json')  // 加载地图源、Token
  → createApp(App) → mount('#app')
  → DTGlobe.vue onBeforeMount
    → DTScopeEngine.getInstance()   // 初始化 Cesium Viewer
    → loadMap(viewer)               // 加载影像底图
  → Dashboard.onMounted
    → initSceneData()               // 加载中线、隧道、地形、围岩、工点
```

### 场景切换流程

```
用户点击场景/数据面板按钮
  → handleSelectScene(sceneKey)
    → 爆破 → 打开 BlastStation 浮窗
    → 施工场景 → 相机 flyTo + 打开双侧面板
      → Lining / Vent / Dispatch 组件
  → handleLayerSelect(layerKey)
    → activateGeoModel(key)         // 加载体数据/GLB + 飞相机
    → 打开对应场景面板 (BLDZ/ZZMSM/DZLD/AHD/DBH/TSP/TEM)
```

### 后端处理流程

```
POST /api/process/:modelKey  (上传文件)
  → multer 保存到 uploads/{jobId}/
  → runProcessingScript(jobId, modelKey)
    → spawn EXE/Python → 解析 stdout JSON 进度 → 更新内存状态

GET /api/process/status/:jobId (轮询进度)
  → 返回 { status, progress, message }
```

| 模型 Key | 中文名 | 数据类型 | 渲染方式 |
|----------|--------|---------|---------|

## 设计模式与亮点

- **单例模式**: `AppConfig`、`DTScopeEngine` 通过 Proxy/静态属性确保全局唯一实例
- **代理模式**: Vite 开发代理 `/api` → `localhost:3000`，前后端解耦
- **异步初始化**: Viewer 就绪检测用 `setTimeout` 轮询 + `getViewer(callback)` 模式
- **Cesium 外部化**: 通过 `vite-plugin-cesium` + `viteExternalsPlugin` 避免打包 30MB+ JS
- **地形透视**: 利用 `globe.translucency.frontFaceAlphaByDistance` 实现近看隧道、远看山体效果
- **AI Agent 集成**: 设计了完整的 Claude Tool Use 体系 (5 个工具)，支持流式 SSE + 平台操作映射，目前暂未启用
- **任务队列**: 后端用内存 Map 管理异步处理任务，stdout JSON 契约式通信，支持 EXE/DLL/Python 三种执行方式
