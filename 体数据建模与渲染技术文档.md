# 体数据建模与渲染技术文档

## 隧道施工数字孪生平台 — 地质体数据可视化

---

## 1. 概述

本项目采用 **双轨渲染体系** 实现隧道地质体数据的三维可视化：

| 渲染方式 | 适用场景 | 核心引擎 |
|----------|---------|---------|
| **WebGL 体渲染**（光线投射） | 富水带、破碎带、TSP 反演、瞬变电磁 | ShareVolume (自研) |
| **Cesium 三角网格**（GLB 模型） | 软弱围岩、掌子面素描、GPR、超前水平钻 | Cesium Model Primitive |

两种方式均在 Cesium 地球场景中进行空间对齐，通过统一的 `GeoModelController` 调度。

### 3.4 核心算法：GPU 光线投射 (Ray Casting)

光线投射完全在 Fragment Shader 中实现 ([index.html:160-290](../index.html#L160-L290))：

```
对每个屏幕像素:
  1. 屏幕坐标 → NDC → 裁剪空间 → 视线空间 → 模型空间
     → 得到射线方向 rayDirection

  2. 射线-包围盒求交
     → 得到近交点 near 和远交点 far

  3. 计算步长
     stepSize = sqrt(3) / uSamples

  4. 从 near → far 逐点步进:
     for each sample point pos:
       ├─ density = tex3D(pos)        // 3D 纹理采样
       │   ├─ 伪 3D: 从 2D 切片拼图定位 Z 切片
       │   └─ 双线性插值: mix(slice[i], slice[i+1], fract(Z))
       │
       ├─ density = (density - range[0]) / range  // 归一化
       ├─ density = pow(density, uPower)           // 幂次增强
       │
       ├─ color = texture2D(uTransferFunction, density)  // 传递函数映射
       │
       ├─ (可选) 等值面检测:
       │   if crossing isoValue:
       │     二分迭代精确定位 (5 次)
       │     计算梯度 → 法线 → Phong 光照
       │
       └─ front-to-back 合成:
           colour += transmittance * sample_colour
           transmittance *= 1.0 - sample_alpha
           if transmittance < 0.01: break
```

### 3.5 3D 纹理编码方案

由于 WebGL 1.0 不支持原生 3D 纹理，采用 **2D 切片拼图** (Texture Atlas) 方案：

```
体素数据: res = [xres, yres, zres]

切片的 2D 排列:
  slices.x = 向上取整(sqrt(zres))  ← 动态计算宏
  slices.y = 向上取整(zres / slices.x)

采样函数 texture3Dfrom2D(pos):
  Z = pos.z × slices.x × slices.y
  slice = int(Z)              // 当前切片索引
  offset = pos.xy × islices   // 切片内坐标
  A = slice × islices.x       // 切片基地址
  B = (slice+1) × islices.x
  return mix(tex2D(A+offset), tex2D(B+offset), fract(Z))
```

### 3.6 传递函数 (Transfer Function)

传递函数将标量密度值映射为 RGBA 颜色，通过 Canvas 2D 渐变色带实现：

```
密度值 [0, 1]
    │
    ▼
┌──────────────────────────┐
│  Palette (渐变色带)        │
│  Canvas 2D linearGradient │  ← GradientEditor 交互编辑
│  → upload as 1D Texture   │
└──────────────────────────┘
    │
    ▼
fragment shader:
  vec4 value = texture2D(uTransferFunction, vec2(density, 0.5))
```

支持功能：
- 多色标点 (Color Stop) 自由添加/删除/拖拽
- Canvas 2D 渐变色带实时预览
- HSV 颜色选择器编辑单个色标
- 色标数据 JSON 序列化/反序列化
- 预乘 Alpha (Premultiplied Alpha) 模式

### 3.7 等值面渲染

在光线步进过程中检测密度跨越等值阈值：

```
if density crosses isoValue:
  → 二分迭代精确定位交点 (5 次)
  → 计算密度梯度 = (tex3D(x+ε) - tex3D(x-ε), ...)
  → 梯度归一化 = 法线
  → Phong 光照: ambient + diffuse × |dot(normal, lightDir)|
  → 可选: 仅渲染包围盒边界面 (uIsoWalls)
```

### 3.8 Cesium 空间对齐

体数据的本地坐标系统通过 `cesiumConfig` 变换到 WGS84 地理坐标：

```javascript
// GeoModelController.ts 中配置
cesiumConfig: {
  rotate:    [rx, ry, rz],    // 欧拉角旋转 (度), ZYX 顺序
  translate: [tx, ty, tz],     // 平移 (米)
  scale:     [sx, sy, sz]      // 缩放
}

// Volume 构造函数中构建 modelMatrix (ShareVolume01.js:2814-2822):
quat.rotateZ(rotat, rotat, rotate[2] * π/180)
quat.rotateY(rotat, rotat, rotate[1] * π/180)
mat4.fromRotationTranslationScale(modelMatrix, rotat, translate, scale)

// 每帧同步 Cesium 相机 (copyCamera):
viewMatrix = CesiumCamera.viewMatrix × modelMatrix
projMatrix = CesiumCamera.frustum.projectionMatrix
```

## 4. 技术路线二：Cesium GLB 三角网格模型

### 4.1 适用场景

不需要体渲染的地质模型使用 Cesium 原生 Model Primitive 加载。

### 4.2 建模工具链

```
外部建模软件 (Rhino / 3ds Max / SketchUp)
  │
  ▼
OBJ 文件导出
  │
  ▼
npm run convert:obj → obj2gltf → GLB 文件
  │
  ▼
存放至 public/data/{model}/model.glb
  │
  ▼
Cesium.Model.fromGltfAsync({ url, modelMatrix })
```

### 4.3 空间定位方式

**单模型模式** (如软弱围岩)：
```javascript
{
  glbUrl: 'data/weiyan/model.glb',
  tunnelPos: [lon, lat, height],
  glbHeading: 1.5708,   // 模型朝向 (弧度)
  glbRoll: 0,
  glbYRot: -1.5708       // 额外绕 Y 轴旋转
}
```

**多模型沿隧道排列模式** (如掌子面素描)：
```javascript
{
  glbItems: [
    { url: '.../2322196.glb', mileage: 0  },
    { url: '.../2322509.glb', mileage: 10 },
    { url: '.../2322518.glb', mileage: 20 },
  ],
  tunnelPos: [lon, lat, height],  // 参考里程位置
  referenceMileage: 0,            // 参考里程 DK 值
  glbHeading: 1.7467              // 隧道走向 (弧度)
}
```

系统根据里程差自动计算每个模型的经纬度偏移：
```javascript
lon = refLon + (dkDiff × sin(heading)) / metersPerDegLon
lat = refLat + (dkDiff × cos(heading)) / metersPerDegLat
```

## 5. 统一调度层 (GeoModelController)

[GeoModelController.ts](../src/utils/Common/GeoModelController.ts) 是体数据与 GLB 模型的统一调度入口：

```javascript
activateGeoModel(key)
  ├─ _cleanup()              // 清理上一个模型
  ├─ new Previous(viewer)    // 重置体渲染状态
  ├─ initVolume(url, cfg)     // 加载体数据 (如有 volumeUrl)
  ├─ loadGlb(url, pos)       // 加载 GLB 模型 (如有 glbUrl/glbItems)
  ├─ 加载随道参考模型          // finaltunnel1.glb (体数据场景)
  └─ camera.flyTo() + lookAt // 飞相机到预设视角

deactivateGeoModel()
  ├─ _cleanup()              // 移除 GLB primitives + 隧道模型
  ├─ clearVolume()            // 移除 WebGL Canvas + 释放资源
  └─ camera.lookAtTransform(IDENTITY)  // 释放相机锁定
```

## 6. 渲染循环机制

```javascript
// index.js: drawVolume()
const loopRender = () => {
  requestAnimationFrame(loopRender);
  if (Previous.SVData[1]) {   // Volume 实例
    Previous.SVData[1].draw(); // 增量渲染: 仅在相机变化时重绘
  }
};
loopRender();
```

体渲染采用 **增量渲染策略**：每帧检测 Cesium 相机矩阵是否变化，仅在相机移动或参数变更时才触发 GPU 着色，降低闲置功耗。

## 7. 数据格式规范

### 7.1 JSON 配置文件结构 (LavaVu 兼容)

```json
{
  "properties": {
    "background": "rgba(0,0,0,0)",
    "nogui": true
  },
  "objects": [{
    "name": "volume",
    "volume": {
      "url": "volume.png",
      "res": [256, 256, 256],
      "scale": [1.0, 1.0, 1.0],
      "autoscale": true,
      "mindensity": 0.0,
      "maxdensity": 1.0
    },
    "samples": 256,
    "density": 10.0,
    "power": 1.0,
    "isovalue": 0.0,
    "isowalls": false,
    "isoalpha": 0.75,
    "tricubicfilter": false,
    "usecolourmap": false,
    "slices": { "properties": { "X": 128, "Y": 128, "Z": 128, "show": false } },
    "cesium": {
      "rotate": [0.0, -1.5, 168.5],
      "translate": [25, 45, -50],
      "scale": [0.0065, 0.00921, 0.00921]
    }
  }],
  "views": [{
    "translate": [0, 0, 4],
    "rotate": [0, 0, 0],
    "axes": false,
    "border": false
  }],
  "colourmaps": [{
    "colours": [
      {"position": 0.0, "colour": "rgba(0,0,255,0)"},
      {"position": 0.5, "colour": "rgba(0,255,0,128)"},
      {"position": 1.0, "colour": "rgba(255,0,0,255)"}
    ]
  }]
}
```

### 7.2 体素纹理格式

- **格式**: PNG 单通道 (LUMINANCE)，8-bit 灰度
- **编码**: 所有 Z 切片水平/垂直平铺在一张 2D 图片中
- **切片排列**: `slices.x` 列 × `slices.y` 行，从上到下、从左到右
- **大小**: width = `xres × slices.x`, height = `yres × slices.y`

## 8. 性能策略

| 策略 | 说明 |
|------|------|
| **增量渲染** | 相机不动时不触发 GPU 重绘 |
| **高低质量切换** | 交互时采样数减半 + 关闭 Tricubic 滤波 |
| **延迟高质量渲染** | `delayedRender(250ms)`: 拖动时低质量实时，松手后 250ms 切高质量 |
| **移动端降级** | 检测 `screen.width ≤ 760` 或 UA 判断，自动禁用交互旋转 |
| **早期终止** | 累积透明度 `T < 0.01` 时 break 光线步进 |
| **Cesium 外部化** | `vite-plugin-cesium` 避免打包 30MB+ Cesium JS |
